## Summary

Blair identified and fixed a UI flicker bug in `ChatViewProvider._sendConfigStatus()`. The preliminary configStatus message was hardcoding `hasAuth: false`, causing the welcome/setup screen to briefly appear on every auth poll and config change—even on fully configured extensions.

## Work Done

- Added `_lastKnownHasAuth` field to `ChatViewProvider` to cache auth state between async checks
- Changed preliminary configStatus to use cached auth instead of hardcoded false
- Preserved first-run behavior by defaulting `_lastKnownHasAuth` to `false`
- Final configStatus still arrives after real async check, correcting any stale cache
