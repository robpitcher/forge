## Key Design Decisions

1. **Error handling**: `session.abort()` wrapped with `.catch()` to handle already-ended sessions
2. **Concurrent abort**: `Promise.all()` parallelizes aborts during deactivation for speed
3. **Separate removeSession()**: Lightweight Map deletion kept for test teardown scenarios
4. **Edge case guard**: Undefined session guard prevents race conditions in Map iteration

## Rationale

- **No memory leaks**: Sessions properly aborted and removed on error or deactivation
- **Graceful deactivation**: Extension can deactivate mid-stream without errors
- **Testable lifecycle**: `getSessionCount()` allows verification in tests
- **Foundation for future**: Enables periodic stale session cleanup (post-MVP)
