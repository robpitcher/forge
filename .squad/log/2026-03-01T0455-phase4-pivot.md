# Session Log: Phase 4 Tool-First Pivot

**Date:** 2026-03-01T04:55Z  
**Lead:** MacReady  
**Participants:** Rob Pitcher (stakeholder)  
**Outcome:** Pivot approved; issues #28, #85, #86, #87 being updated

## What Happened

Rob questioned the Phase 4 scope: "we should just be exposing those tools, right?" This prompted MacReady to analyze the SDK's actual capabilities.

## Key Finding

The Copilot SDK v0.1.26 includes a full agentic tool system with built-in categories (`shell`, `read`, `write`, `url`, `mcp`) and configuration controls (`availableTools`, `excludedTools`, `customAgents`, `systemMessage`, `onPermissionRequest`, etc.). Forge already has tool approval UX working.

Hardcoded slash commands (#85) would add only string concatenation — a less powerful abstraction than what the SDK provides natively.

## Decision

**Phase 4 pivot: tools-first UX, not slash commands.**

- **#85 (Slash commands):** Close — won't implement. Model handles explain/fix/test natively.
- **#86 (Code actions):** Rework — keep concept, drop slash command dependency. Use native agent capabilities.
- **#87 (Conversation history):** Keep as-is.

### New Phase 4 Focus

1. Tool UX polish (streaming output, tool names, arguments)
2. Tool control settings (expose `availableTools`/`excludedTools`)
3. Conversation history (unchanged)
4. Code actions (reworked to use native capabilities)
5. System message customization
6. MCP server support (stretch)

## Impact

- Reduced implementation scope
- Better air-gap story (tool control defaults exclude `url`)
- Aligns with SDK design; opens door to MCP integration

## Next Steps

MacReady updating GitHub issues to reflect new direction.
